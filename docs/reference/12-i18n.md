## TL;DR

- Single source: `master_plan_it/master_plan_it/translations/it.csv`
- Filename must match Frappe Language code (e.g., `it.csv` or `it-IT.csv`)
- DocType JSON auto-translates; code strings require explicit `_()` / `__()`

## Marking Strings

| Context | Syntax | Example |
|---------|--------|---------|
| Python | `_(...)` | `_("Welcome {0}").format(name)` |
| JS | `__(...)` | `__("Welcome {0}", [name])` |
| Jinja | `{{ _(...) }}` | `{{ _("Total") }}` |

## Literal String Rules

- Use literal strings only (no variables)
- Positional placeholders: `{0}`, `{1}`
- No concatenation, multiline, trailing spaces
- Context for ambiguous terms: `_("Change", context="Switch")`

## CSV Format

```csv
source_string,translated_string,context
"Welcome {0}","Benvenuto {0}",
"Change","Modifica","Switch"
```

- 3 columns: source, translation, context (empty if not needed)
- Quote values with commas
- Sort by source_string then context

## What NOT to Translate

- DocType names, fieldnames, module names
- Route paths, DB keys
- App/product names (unless in glossary)

## Verification

1. Set user language to Italian
2. Confirm `translations/it.csv` exists
3. Hard-refresh browser
4. If stale: `bench --site <site> clear-cache`
5. If still stale: restart bench services
6. Only rebuild assets if custom JS changed: `bench --site <site> build`

## Common Issues

| Symptom | Cause |
|---------|-------|
| String not translated | Not wrapped in `_()` / `__()` |
| Still stale after clear-cache | Filename mismatch with Language code |
| Wrong file location | Must be under app package, not elsewhere |
